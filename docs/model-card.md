# Model Card

This Model Card documents the current baseline model for the Part 3 FAIR Data Science package. The generated evaluation outputs have reserved TUWRD DOI `https://doi.org/10.70124/4wqg-7oc34`. The Model Card should still be updated with the final TUWRD model DOI after the model artefact is deposited.

## Model Description

This project trains a baseline machine learning model for classifying elevated nitrogen dioxide (NO2) measurements across Austrian monitoring stations. The input data consists of hourly NO2 measurements from the European Environment Agency for 2024. The trained artefact is `outputs/models/no2_air_quality_classifier.joblib`.

The current model uses `RandomForestClassifier` from scikit-learn with `n_estimators=100`, `max_depth=12`, `class_weight="balanced"`, and `random_state=42`. The elevated-NO2 target is defined as `Value >= 40 ug/m3`. This threshold is used as an educational classification label and is not presented as an official regulatory decision rule.

## Intended Use

The intended use of the model is to support reproducible analysis of NO2 air-quality classes in the selected Austrian monitoring-station dataset. It is intended for educational open-science demonstration, FAIR data stewardship practice, and reproducible environmental data analysis. It should be used together with the documented data source, preprocessing steps, and evaluation metadata.

## Out-of-Scope Uses

The model is not intended for regulatory air-quality enforcement decisions. It is not intended to provide real-time public-health warnings or replace official monitoring and reporting by environmental authorities. It should not be applied to other pollutants, countries, years, or station networks without validating that the data distribution and class definitions are compatible.

## Training Data

The training data is derived from EEA NO2 hourly measurement files stored in `data/raw/`. The current raw dataset contains 16 station-specific Parquet files and 140,160 hourly observations. The repository DOI is `https://doi.org/10.5281/zenodo.20432795`, and the DBRepo database is `https://test.dbrepo.tuwien.ac.at/database/ed890fa1-154c-4a66-8529-4088c97f68db` with DOI `https://doi.org/10.82556/3zan-dn41`.

The train/test split is stratified with an 80/20 split. The final run used 112,128 training rows and 28,032 test rows. There are 9,352 elevated-NO2 rows in total, which is about 6.67% of the dataset.

## Evaluation Results

The evaluation results below are generated by `scripts/train_no2_classifier.py`. The measured NO2 value is used to create the target label, but it is not used as an input feature. The model uses time, station, validity, verification, and data-capture metadata as features.

| Metric | Value | Notes |
| --- | --- | --- |
| Accuracy | 0.8005 | Overall test-set accuracy |
| Precision | 0.2107 | Precision for the elevated-NO2 class |
| Recall | 0.7251 | Recall for the elevated-NO2 class |
| F1 score | 0.3266 | F1 score for the elevated-NO2 class |

The evaluation files are stored in `outputs/results/model_metrics.json`, `outputs/results/confusion_matrix.csv`, `outputs/results/test_predictions.csv`, and `outputs/figures/confusion_matrix.png`.

The generated-output deposit contains these evaluation files and has reserved TUWRD DOI `https://doi.org/10.70124/4wqg-7oc34`. It was prepared as a dataset deposit for the generated outputs, not as the trained-model deposit.

## Limitations

The model is limited by the coverage of the selected 2024 EEA station files and may not generalise outside the included monitoring points. The raw data currently covers NO2 only, so the model does not represent broader multi-pollutant air quality without additional data. Because the measured NO2 value is excluded from the input features to avoid target leakage, the model is mainly a baseline that learns station/time patterns rather than a direct concentration classifier. Its recall is higher than its precision, so it should be interpreted carefully.

## Ethical Considerations

Air-quality analysis can influence public interpretation of environmental risk, so the model output must be presented with clear uncertainty and scope limitations. The project should avoid implying official authority or regulatory validity unless the results are independently validated. The model should be documented transparently so that users can inspect the input data, preprocessing, assumptions, and evaluation results.
